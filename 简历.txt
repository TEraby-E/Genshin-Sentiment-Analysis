项目简介：

独立开发端到端的玩家舆情分析与内容运营平台，覆盖原神 B 站生态的 37,802 条视频元数据、40.8 万条评论与 1,607 条官方动态（2024.07–2025.11）。项目从“可解释、可复现、有统计依据”的传统机器学习与统计推断基线起步，构建舆情监控与内部工具看板；后期为突破口语化、反讽、社区黑话场景下的语义瓶颈，逐层引入大模型 API、RAG 检索增强、知识蒸馏与 LoRA 微调，并以一个“算力感知”的智能路由 Agent 把多条打标能力按成本-精度统一调度，最终落地“本地调用、云端算力”的弹性部署，形成从数据抓取、统计预警到 AI 深度研判的全链路工程闭环。

技术栈：

数据分析与统计：pandas, numpy, scipy, scikit-learn（随机森林 / 字符级 TF-IDF + 逻辑回归）, 双比例 z 检验, 置换检验, 滚动窗口 z-score, 功效分析

大模型与 RAG：DeepSeek-v4-pro API（OpenAI 兼容）, Qwen2.5-7B（QLoRA 微调）, LLaMA-Factory, peft / transformers / bitsandbytes, 本地向量库（numpy 内存 / ChromaDB）, 混合检索（哈希/语义嵌入 + BM25）

智能体编排与部署：手写多轨道路由 Agent（检索-推理-校验三角 + 成本阶梯升档）, vLLM（OpenAI 兼容推理端点）, Docker / docker-compose, Streamlit

数据工程与规范：requests + B 站 WBI 动态签名, uv, pytest（fake client/embedding，100+ 单测）, ruff, mypy, GitHub Actions CI

技术亮点：

统计严谨的舆情预警基线：把 40.8 万条评论的负面情绪按月聚合，用双比例 z 检验确认版本更新期的情绪峰值在统计上显著（峰值 21.5% vs 基线约 10.7%），并用滚动窗口 z-score 替代固定倍数阈值以适应舆情基线随版本的缓慢漂移；数据加载阶段做列存在性、空值率、日期解析率三类契约校验，避免脏数据静默流入下游。配套同月匹配 + 置换检验做跨界联动的因果推断，在极小样本（n=6）下诚实报告效应量与置信区间，而非用大样本假设硬套。

数据驱动的模型选型论证：先用关键词方面分类器在 40 万真实评论上实测覆盖率仅 18.2%，用量化证据证明关键词规则在口语化/表情符号场景的盲区，以此驱动“引入 LLM 做语义级打标”的决策，而非凭直觉。

AI 文本工作流（调用主流模型 API）：text_pipeline.py 实现「自动化清洗 → 语义级归类 → 舆情总结」三段式工作流，调用 DeepSeek（OpenAI 兼容协议，可一键切换到任意兼容服务）做：规则清洗（去链接/@/回复前缀/重复字符）→ LLM 批量打标（情感极性 + 多方面分类，强制 JSON 输出 + 指数退避重试 + 非法标签兜底 + 单批失败隔离）→ LLM 舆情总结（核心议题、代表性原声、词云关键词、可执行运营建议），密钥从 .env 读取、绝不入库。

RAG 反幻觉，对抗社区黑话误判：从数据集异步构建本地「梗 & 设定词典」向量库，调用大模型前先用混合检索器（稠密哈希/语义嵌入 + 稀疏 BM25，归一化后加权融合）拦截评论中的领域黑话、检索其释义并注入系统提示，在分类前为模型补足领域语义支撑，缓解按字面误判。向量库默认是零依赖的纯 numpy 内存实现（可离线、CI 友好），需要持久化时再切 ChromaDB 本地模式；嵌入默认确定性哈希、可选 sentence-transformers 语义嵌入。

数据飞轮与知识蒸馏：以 DeepSeek 作为高精度标注源对小样本生成监督信号，经置信度规则（合法标签 + 非空判断依据 + 最小长度）筛选出高置信监督样本集，在其上训练字符级 TF-IDF + 逻辑回归的轻量下游分类器，完成从大模型到本地模型的知识蒸馏，实现离线、免 API、毫秒级的全量推理；以与大模型标注的标签一致率（agreement rate）作为蒸馏质量的量化评估指标。进一步将高置信标注按主题分层抽样、切出留出评估集，转为 LLaMA-Factory 监督微调数据集，在 RTX 4090（24GB）GPU 上以 QLoRA 对 Qwen2.5-7B 做 LoRA 微调，并用留出集评估（准确率 / 宏 F1 + 反讽错例分析）驱动针对性补样的增量迭代，得到更适配领域语义与社区黑话、且支持离线推理的本地分类器（重依赖全部延迟导入）。

算力感知的动态智能路由（Router Agent）：把关键词、蒸馏、本地/云端 LoRA、RAG-DeepSeek 等多条能力统一成可调度的“轨道”，由 RouterAgent 用零成本的离线难度画像（黑话密度、反讽标记、长度）决定算力投入——简单评论走零成本轨道秒级处理，高争议或反讽的难句才进入「检索 → 推理 → 校验」三角研判，校验不通过则沿成本阶梯（关键词 → 蒸馏 → 本地/云端 LoRA → RAG-DeepSeek）自动升档重判，将高成本推理路径仅分配给高难度样本。校验者默认是零成本的规则实现（情感词典极性冲突检测 + 结果完整性），也可换成大模型评审员；拿不到 API/GPU/模型的轨道自动跳过，离线优雅退化。该编排为手写实现、零额外框架依赖，逻辑透明可调试。

本地调用 + 云端算力的弹性部署：在云 GPU 上微调后用 vLLM 把 Qwen 起成 OpenAI 兼容端点、经隧道映射出公网地址，本地通过 ServedLLMClient 复用统一的 OpenAI 协议并强制改写模型名，零改动即可让路由新增一条云端 LoRA 轨道、且不消耗 DeepSeek 额度。全栈容器化（CPU 应用镜像 + GPU 推理服务的 docker-compose profile），并提供一键 healthcheck 自检 DeepSeek / RAG / 蒸馏 / 云端·本地 LoRA / 路由 / 看板的可用状态。

全栈数据工程与运营赋能：逆向复刻 B 站 WBI 动态签名算法，构建带请求限速、磁盘缓存与优雅降级的全站监控流水线（排行榜快照对比、竞品热度聚合）；用随机森林做爆款预测（AUC ≈ 0.60，并据此得出“爆款由内容质量主导而非发布技巧”的业务结论），结合 UP 主生命周期分群识别高价值唤回对象；最终通过 Streamlit 六工具看板把作品打标、作者分群、爆点预测与舆情 AI 总结一键工具化。全程 100+ 个单元测试（合成 fixture + fake LLM/HTTP/嵌入，不依赖真实数据、网络或 GPU）、ruff + mypy + GitHub Actions CI 三件套保障工程质量。
