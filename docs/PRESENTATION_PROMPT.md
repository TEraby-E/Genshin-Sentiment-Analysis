## 提示词正文

你是一名擅长用数据讲故事的咨询顾问，现在要帮我把一个数据分析项目整理成一份**面向 HR / 非技术面试官**的演示文稿（PPT），用于游戏公司数据分析岗位的实习/校招面试展示。

### 受众与基调
- 受众是 HR 和业务面试官，**不是数据科学家**，看不懂模型细节，但需要看懂"做了什么、为什么有价值、得出了什么结论"。
- 语言要简单直白，避免堆砌术语；每页只讲一件事；多用数字和一句话结论，少用大段文字。
- 整体基调：专业、严谨、但不故作高深——重点展示"遇到模糊或有局限的结果时，如何用统计方法和业务视角把它转化为可信、可执行的结论"，这是比"做了多少分析"更重要的加分项。

### 项目背景
项目名称：《原神 B 站玩家生态与舆情分析》
数据来源：Kaggle 公开数据集（B 站视频、评论、原神官方帖子）
数据规模：视频 37,802 条 / 评论 408,427 条 / 官方帖子 1,607 条
项目性质：个人独立完成的端到端数据分析项目，从数据清洗到统计建模到可视化全流程自己搭建

### 请按以下大纲生成 12–14 页幻灯片，每页给出：标题 + 3–5 条要点（每条不超过20字）+ 建议配图说明

**第1页 封面**
项目标题、一句话价值主张（例如："从公开数据出发，用统计方法把模糊的舆情和市场信号变成可执行的业务结论"）、姓名/日期

**第2页 为什么做这个项目**
游戏内容运营和市场团队每天要回答的三个问题：
1. 玩家最关注什么内容？
2. 负面情绪什么时候真的在恶化？
3. 营销动作（如跨界联动）有没有真实效果？
本项目用真实公开数据，端到端回答这三个问题

**第3页 数据规模总览**
三类数据的规模对比（视频/评论/官方帖子），配图：柱状图（对数刻度）
强调评论量是视频量的十倍以上，体量足够支撑统计检验

**第4页 数据质量是分析的起点**
约35%的视频是泛娱乐噪声内容，已识别并剔除
长尾分布下用中位数而非均值衡量内容表现（避免被少数爆款误导）
加载阶段做数据契约校验，数据格式变化时第一时间报错

**第5页 玩家最关注什么内容**
角色专题、二创内容、攻略解析是社区内容生态核心
配图：内容主题热度排行横向柱状图

**第6页 舆情监控：从经验阈值到统计显著性**
传统方法：负面占比超基线1.5倍就预警（没有统计依据）
本项目方法：双比例z检验 + 滚动窗口异常检测
真实结果：峰值月份负面占比21.5% vs 基线10.7%，统计显著（p<0.05）
配图：舆情月度走势折线图，标出峰值和预警线

**第7页 舆情不只是"差"，更要知道具体差在哪**
在粗粒度情感分类之上，识别六个具体方面（剧情/抽卡/活动/数值机制/运营/社交）
"运营"和"社交"方面负面情绪更值得优先处理（更容易直接调整）
配图：各方面提及量与负面占比的横向柱状图

**第8页 创作者生命周期：资源该投向谁**
将UP主划分为单发/成长期/稳定期/沉寂期
反直觉发现：沉寂期创作者历史播放表现反而优于稳定期 → 是潜在的"唤回"目标，不该只盯着当前活跃的创作者
配图：四个阶段的创作者数量与播放表现对比表/柱状图

**第9页 跨界联动：营销动作到底有没有用**
用"同月匹配+置换检验"的因果推断方法（控制版本节奏等混杂因素）
真实结果：跨界联动帖子相对同期普通帖子点赞数提升532%，统计显著（p<0.001）
诚实说明：联动事件样本量小（n=6），结论方向可信但置信区间较宽
配图：联动 vs 普通帖子互动量对比柱状图

**第10页 爆款预测：靠运营技巧还是靠内容本身**
随机森林预测模型，AUC仅0.604（接近随机）
解读："模型效果不够好"本身就是结论——发布技巧很难预测爆款，说明爆款更可能由内容质量决定
配图：特征重要性排行图

**第11页 A/B实验评估能力展示**
搭建了可复用的A/B实验评估框架：样本量计算 + 显著性检验
用模拟数据验证方法正确性（因为公开数据集没有真实随机实验），可直接套用到真实营销活动的效果评估
配图：示意图（对照组 vs 实验组转化率对比 + 置信区间）

**第12页 工程能力：让结论可信、可复现**
36+单元测试（不依赖真实数据即可在CI跑通）
ruff静态检查 + mypy类型检查 + GitHub Actions持续集成
数据契约校验，数据格式变化时主动报错而非悄悄出错

**第13页 总结：这个项目证明了什么**
三个核心结论（内容生态/舆情预警/爆款归因）
更重要的是方法论能力：从"经验判断"到"统计检验"，从"模糊结果"到"可执行结论"
这正是数据分析/营销策略岗位日常工作中最常遇到、也最考验功底的部分
